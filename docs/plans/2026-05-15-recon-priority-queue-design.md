# 远勘优先目标队列设计

## 背景

远勘已经完成任务台化，但公网结果区域仍然存在信息重复：研判概览、逐目标结论、重点目标都在解释同一批目标。用户希望整体更实用、看得爽，因此本轮聚焦把公网结果改成“优先目标队列”。

## 目标

- 只修改远勘智能体
- 减少公网结果重复卡片
- 默认一行一个目标，展示优先级、目标、核心结论和下一步
- 点击目标后再看重点目标详情
- 保留报告、技术细节和深度远勘能力

## 设计

公网远勘结果顺序调整为：

1. 任务台
2. 过程/统计折叠区
3. 优先目标队列
4. 重点目标详情
5. 技术细节折叠区
6. 报告草稿

取消单独的研判概览强卡片，把概览内容融入目标队列和任务台。

## 队列信息结构

每行目标展示：

- 优先级标签
- 目标 URL
- 网站用途一句话
- 关键线索一句话
- 下一步动作
- 少量风险标签

## 技术策略

- 新增 `buildPriorityQueueLabel` helper，统一优先级标签
- 在 `PentestPanel` 中新增 `renderPriorityTargetQueue`
- 移除公网结果中的重复研判概览区
- 将逐目标结论卡改成更紧凑的队列行

## 验证标准

- 测试通过
- TypeScript 诊断无错误
- 构建通过
- 公网结果默认信息密度明显降低
