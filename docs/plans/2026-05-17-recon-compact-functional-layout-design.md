# 远勘智能体强压缩功能型布局设计

## 背景

当前远勘智能体的问题不是能力入口不够，而是介绍、标签、说明和大按钮占用了过多首屏空间，导致真正的研判结果、优先目标和下一步动作显示比例偏小。用户明确要求：减少大标签介绍和无用标签，只保留功能，让结果成为主视觉。

## 设计目标

- 首屏优先展示结果，不优先展示介绍。
- 删除或压缩大 Hero、大任务台、大状态说明卡。
- 输入、运行、导出、升级深度远勘等操作集中到紧凑工具栏。
- 公网远勘以优先目标队列和重点目标详情为主。
- 深度远勘以结构化结论和继续追问为主。
- 过程、统计、报告原文、材料准备全部默认折叠。
- 保留现有远勘能力，不影响服务器取证、通用智能体和内网扫描链路。

## 总体布局

### 公网远勘

默认结构调整为：

1. 紧凑操作栏
   - 模式名
   - 目标输入
   - 备注折叠
   - 深度选择
   - 开始/停止/导出/带入深度远勘

2. 结果主体
   - 优先目标队列占主要宽度
   - 重点目标详情紧随其后或在宽屏右侧展示
   - 空状态只提示如何开始，不展示长说明

3. 折叠辅助区
   - 过程轨迹
   - 批量统计
   - 报告草稿和原文

### 深度远勘

默认结构调整为：

1. 紧凑操作栏
   - 模式名
   - 目标输入
   - 开始/停止/导出

2. 主结果区
   - 结构化结论
   - 证据依据
   - 下一步动作
   - 继续追问输入条

3. 折叠辅助区
   - 材料准备
   - 智能体过程
   - 对话记录
   - 报告草稿

## 模块策略

### 删除或替换

- `renderWorkspaceHero`：不再用于公网远勘和深度远勘，改成紧凑栏。
- `renderTaskDesk`：从大卡片压缩为单行行动条。
- `renderAgentTrace`：从四张大阶段卡压缩为轻量时间线。
- 大量 `renderStatusBanner`：在结果详情中改成更薄的信息行。

### 保留但降级

- 过程轨迹：保留，默认折叠。
- 报告草稿：保留，默认折叠。
- 输入材料：深度远勘保留，但默认折叠。
- 风险标签：每个目标最多显示 1-2 个。

## 视觉原则

- 不再让标签、标题、说明文本形成大块视觉占位。
- 卡片边距从大面积 `p-5/p-6` 降到 `p-3/p-4`。
- 操作按钮改为紧凑尺寸。
- 结果列表和结论行使用高密度排版。
- 只保留必要的颜色用于风险、状态和主操作。

## 验证标准

- 公网远勘首屏不再出现大 Hero。
- 公网远勘结果出现后，优先目标队列和重点目标详情占据主要空间。
- 深度远勘首屏不再出现大 Hero 和大任务台。
- 过程、统计、报告、材料默认折叠。
- 构建、测试、类型检查通过。
