# 远勘智能体决策优先与多层过程设计

## 背景

用户反馈远程勘查结果和界面仍然眼花缭乱，线索与信息太多，看完不知道下一步应该做什么。用户希望整体更接近 Codex 或 Claude Code：不仅给结果，还能看到多层调用、阶段过程和可审计的工作链路。

## 资料结论

Codex / Claude Code 类工作台的关键体验不是展示更多字段，而是展示可理解的 Agent 循环：

- 收集上下文
- 调用工具执行动作
- 观察工具结果
- 归纳判断
- 验证并给出下一步

因此远勘智能体应从信息展示页转成决策优先的 Agent 工作台。

## 设计目标

- 只修改远勘智能体
- 第一屏先告诉用户下一步做什么
- 多层调用过程可见，但不暴露模型私有思维链
- 把大量线索降级为依据、补充信息和完整原文
- 保留公网远勘、深度远勘、内网扫描的边界

## 公网远勘

结果页顺序调整为：

1. 下一步行动卡
2. 远勘过程轨迹
3. 批量统计和整体概览折叠区
4. 逐目标结论
5. 重点目标详情
6. 技术细节折叠区
7. 报告草稿

行动卡由目标风险分、线索数量、可访问状态、首要可调证线索生成，避免用户先面对大量字段。

## 深度远勘

结果页顺序调整为：

1. 下一步行动卡
2. 智能体过程轨迹
3. 侦查输入与材料准备折叠区
4. 结构化结论
5. 对话记录
6. 报告草稿

过程轨迹只展示阶段摘要和工具层级，不展示隐私思维链。

## 技术策略

- 在 `src/lib/reconWorkspace.ts` 新增行动决策和 Agent 轨迹 helper
- 在 `src/components/PentestPanel.tsx` 新增行动卡和过程轨迹渲染
- 保持当前单列主内容结构
- 通过现有 `scripts/recon-workspace.test.mjs` 增加 helper 回归测试

## 验证标准

- 公网远勘结果第一屏能直接看到下一步动作
- 深度远勘能看到 Agent 阶段轨迹
- 大量统计、输入和原始内容默认下沉
- 测试通过
- TypeScript 诊断无错误
- 构建通过
