# 远勘智能体任务台化设计

## 背景

远勘智能体已经具备行动卡、过程轨迹、逐目标结论和报告能力，但首屏仍然存在多个强视觉区块同时出现的问题。用户希望继续整体优化，减少眼花缭乱，让系统更像一个可以直接执行下一步的工作台。

## 目标

- 只修改远勘智能体
- 首屏只突出主行动、主按钮、当前状态
- 过程轨迹、统计概览、材料输入和原始信息默认下沉
- 保留 Codex / Claude Code 式多层过程，但不让过程抢主视觉
- 让用户一眼知道现在该点什么

## 总体设计

将结果页从“行动卡 + 过程卡 + 结果列表”的平铺结构，收敛为任务台：

1. 主行动台
2. 过程折叠区
3. 次级线索区
4. 详细结果区
5. 报告区

## 公网远勘

公网结果首屏显示：

- 推荐动作
- 优先目标
- 主按钮：带入深度远勘或查看重点目标
- 当前优先级

过程轨迹、批量统计和研判概览默认折叠。逐目标列表保留，但默认更紧凑。

## 深度远勘

深度远勘首屏显示：

- 当前建议动作
- 主按钮：开始深度远勘、继续追问或导出报告
- 当前状态

智能体过程和材料输入默认折叠，结构化结论保持主阅读位置。

## 技术策略

- 新增任务台动作 helper，统一根据状态选择主按钮文案和提示
- 在 `PentestPanel` 中新增 `renderTaskDesk` 渲染函数
- 公网远勘和深度远勘分别接入任务台
- 将过程轨迹放入折叠区
- 保留现有测试并新增任务台 helper 测试

## 验证标准

- 首屏信息量减少
- 主按钮更明确
- 过程轨迹不再抢占首屏
- 测试通过
- TypeScript 诊断无错误
- 构建通过
