# 远勘详情轻量化设计

## 背景

优先目标队列已经让公网结果更清晰，但点击目标后的重点详情仍然较重。深度远勘结果区也仍然连续展示结构化结论、对话记录和报告草稿，形成二次信息压力。

## 目标

- 只修改远勘智能体
- 公网重点目标详情改成三行式轻详情
- 深度远勘默认先展示最终结论
- 对话、原文和报告默认后置或折叠
- 保留完整信息追溯能力

## 公网远勘

重点目标从大卡片改为轻详情：

1. 结论摘要
2. 核心依据
3. 建议动作

站点类型、业务画像、IP、风险分等字段降为小型元信息条。技术细节继续折叠。

## 深度远勘

深度结果顺序调整为：

1. 最终结论
2. 依据和后续建议
3. 对话记录折叠
4. 报告草稿折叠

默认只看结论，不被对话和报告抢主视觉。

## 验证标准

- 测试通过
- TypeScript 诊断无错误
- 构建通过
- 公网详情更短
- 深度远勘默认阅读压力更低
