# 远勘工作台整体舒适度优化设计

## 背景

当前远勘板块的功能结构已经完成第一轮重构：

- `公网远勘`
- `深度远勘`
- `内网扫描`

但从“大项目完成度”和“整体舒适度”来看，当前仍存在以下问题：

- 三个模式虽然已分层，但页面骨架还不统一，切换后像三个独立页面而不是同一个工作台。
- 第一屏信息节奏仍不够稳定，有的模式更偏表单，有的模式更偏结果，视觉心智不一致。
- 状态、反馈、空态、完成态的表达方式分散，缺少统一的产品级反馈系统。
- 卡片语言、间距、标题层级和信息主次还不够统一，导致整体精致度不够。

本轮目标不是继续加功能，而是在不影响既有能力边界的前提下，把远勘板块打磨成一个更完整、更舒服、更像成熟产品的工作台。

## 目标

- 统一 `公网远勘`、`深度远勘`、`内网扫描` 的页面骨架。
- 统一第一屏的信息节奏，强化“主动作 + 当前状态 + 核心结果”的阅读路径。
- 统一卡片、标题、状态、空态、按钮的视觉语言。
- 让远勘模块从“能用”提升到“整体顺、看着不累、切换有一致性”。
- 保持既有功能边界不变，不影响服务器取证、通用智能体和内网扫描底层逻辑。

## 设计原则

### 1. 先统一工作台，再细化单页

本轮优先解决三块页面“不像一个系统”的问题，而不是继续只优化某一个子页。

### 2. 第一屏只做三件事

每个模式的第一屏都只突出：

- 当前模式定位
- 当前可执行的主动作
- 当前最重要的状态或结果

次级信息全部下沉。

### 3. 默认看结论，细节按需打开

继续保持“少元素、低负担”的方向。技术细节、原始输出、长文本报告，不再抢占第一屏视觉优先级。

### 4. 统一反馈语言

运行中、成功、失败、空状态、待开始，全部使用统一的组件语义和视觉规则，减少页面情绪跳变。

## 信息架构

### 外层统一骨架

`PentestPanel` 统一为四层结构：

1. 顶部工作台头部
2. 模式导航
3. 模式主内容区
4. 全局反馈区

### 顶部工作台头部

所有模式共享同一套头部区域，统一展示：

- 模块名称
- 模块定位说明
- 当前模式状态摘要
- 右侧模式级主动作或辅助动作

这样可以保证用户切换页面时，视觉锚点不变。

### 模式导航

当前的三个切换按钮升级为更像工作台导航的结构：

- 保留 `公网远勘 / 深度远勘 / 内网扫描`
- 增强当前态和 hover 态层次
- 补充简短副标题或状态标签

目标是让切换本身就体现“同一系统下的三种工作模式”。

### 主内容区

三个模式统一成“主区 + 次区”的结构：

- 左侧或上方：主操作区 / 主结果区
- 右侧或下方：次级结果 / 说明 / 技术细节 / 原始输出

虽然具体内容不同，但布局节奏保持一致。

## 各模式落地策略

### 公网远勘

保留现在的“逐目标结论卡”方向，但进一步统一工作台节奏：

- 第一屏：
  - 头部定位
  - 目标输入
  - 采集深度
  - 开始远勘主按钮
  - 当前进度或概览
- 第二屏：
  - 研判概览
  - 逐目标结论列表
  - 重点目标卡
- 第三层：
  - 技术细节
  - 报告草稿

公网远勘的重点是更像“研判工作流”，而不是“数据面板”。

### 深度远勘

深度远勘要从“AI 对话页”进一步升级为“专业工作台”：

- 左侧固定为侦查输入和重点配置
- 右侧统一为：
  - 对话过程
  - 结构化结论
  - 报告草稿

同时增加更稳定的状态反馈，让用户始终知道当前是在：

- 待开始
- 研判中
- 已完成
- 已中断
- 可继续追问

### 内网扫描

内网扫描功能不动，但要视觉纳入同一工作台语言：

- 第一屏只保留连接状态、上传状态、执行动作
- 结构化结果和原始输出下沉
- 与公网/深度远勘共用统一的卡片外壳、标题层级、状态条

这样可以做到“边界清晰，但气质统一”。

## 视觉与交互统一项

### 卡片系统

统一三类卡片：

- 一级卡：Hero / 主操作卡
- 二级卡：功能区卡片
- 三级卡：结果明细卡

统一圆角、边框透明度、阴影层级、内部留白。

### 标题层级

统一为：

- 页面标题
- 分区标题
- 卡片标题
- 辅助说明

避免现在不同模式下字号和间距节奏不一致的问题。

### 状态系统

统一状态语义：

- idle
- loading
- success
- warning
- error

统一展示位置：

- 页面级状态优先在头部或主内容区顶部展示
- 全局 `message` 收敛为更像系统通知条的风格

### 空状态

统一空状态样式和文案语气：

- 公网远勘未开始
- 深度远勘未开始
- 内网扫描无输出

要求看起来属于同一个设计系统。

## 技术实现策略

本轮仍以 `src/components/PentestPanel.tsx` 为主战场。

优先不拆文件，先把结构统一跑通：

- 提炼通用页面头部区域
- 提炼模式导航数据结构
- 提炼统一卡片与状态块渲染函数
- 重排三个模式的布局层级

如果本轮结束后结构明显稳定，再考虑后续拆分：

- `PublicReconWorkspace`
- `DeepReconWorkspace`
- `InternalScanWorkspace`

## 验证方案

- 三个模式切换后，第一屏结构明显统一
- 公网远勘、深度远勘、内网扫描都保留原有核心能力
- 远勘结果与深度远勘对话逻辑不回退
- `PentestPanel.tsx` 无诊断错误
- 构建通过
- 主观体验上：
  - 信息层级更稳定
  - 阅读负担更低
  - 切换页面时不再割裂
