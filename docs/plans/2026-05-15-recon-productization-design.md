# 远勘智能体产品化优化设计

## 背景

远勘智能体已经完成了三轮关键演进：

- 双层能力重构：`公网远勘 / 深度远勘 / 内网扫描`
- 统一工作台外壳
- 摘要层 / 依据层 / 原始层的渐进披露

现在继续做“整体优化”，重点已经不再是继续补结构，而是把远勘智能体推到更像成熟产品成品的状态。

本轮仍严格遵守边界：

- 只修改远勘智能体
- 不改服务器取证
- 不改通用智能体
- 不改其他业务模块

## 本轮目标

将远勘智能体从“结构已经清楚”继续提升到“输入顺手、交互顺滑、报告成品感强”的产品化状态。

本轮同时提升三块：

1. 输入体验
2. 交互完成度
3. 报告成品感

## 设计原则

### 1. 继续低负担，不回到堆信息

本轮不是往页面上继续加块，而是把已有能力变得更顺手、更高级、更像成品。

### 2. 每个模式都要更像完整流程

三种模式都必须形成明确链路：

- 输入
- 执行
- 结果
- 报告

用户不需要思考“接下来该干什么”。

### 3. 先保证可信，再追求惊艳

视觉继续保留未来感，但不能牺牲专业感、结论可信感和阅读效率。

## 优化方向

### 输入体验

#### 公网远勘

- 加强输入框的引导与状态感
- 明确目标数量、当前备注用途、当前采集深度的含义
- 让开始动作更像主流程起点，而不是普通按钮

#### 深度远勘

- 把左侧输入区进一步做成“案件材料录入台”
- 通过分组、辅助说明和主次动作层级，让用户知道：
  - 哪些是必填
  - 哪些是补充材料
  - 哪些是本轮研判重点

#### 内网扫描

- 强化连接、上传、执行的顺序感
- 把关键前提条件更明显地前置展示

### 交互完成度

#### 卡片状态

- 逐目标卡增加更强的选中态
- 折叠区增加更明确的展开感
- Hero、状态条、重点卡之间建立更统一的视觉节奏

#### 过渡与微交互

- 不做花哨动画
- 只增加轻量、专业的过渡：
  - 选中抬升
  - 展开平滑
  - 状态反馈高亮
  - 报告区切换更顺

#### 系统反馈

- 统一空态、进行中、完成、警示、失败的反馈形式
- 让消息更像系统通知，而不是普通文本提示

### 报告成品感

#### 公网远勘

- 将报告区从“原始 markdown 容器”升级为更像正式研判摘要的成品阅读块
- 让用户在导出前就能看到更接近最终成品的样子

#### 深度远勘

- 进一步把深度远勘报告做成“专业研判稿”
- 固定结构：
  - 重点结论
  - 核心依据
  - 可调证信息
  - 后续侦查建议

#### 内网扫描

- 扫描摘要继续强化成“总览结论块”
- 原始输出和明细维持后置，不抢第一阅读层

## 技术实现策略

仍然以 `src/components/PentestPanel.tsx` 为主，不拆到其他业务模块。

必要时仅新增远勘专用的小型 helper，范围限定在：

- `src/lib/reconWorkspace.ts`
- 与远勘智能体直接相关的测试脚本

本轮优先使用：

- 更强的输入分组和辅助提示
- 更统一的成品化结果卡
- 更高级的报告展示块
- 更明确的选中态和局部过渡

## 验证标准

- 只有远勘智能体发生变化
- 公网远勘输入和结果路径更顺
- 深度远勘左录入右研判更像完整产品
- 内网扫描首页继续更清晰，但不改底层逻辑
- 报告区明显更像成品，而不是原始文本容器
- `GetDiagnostics` 无报错
- `npm run build` 通过
