# 远勘工作台拉满版体验设计

## 背景

远勘智能体已经完成两轮核心改造：

- 从旧 `CTF / Lab` 拆分为 `公网远勘 / 深度远勘 / 内网扫描`
- 建立了统一的远勘工作台外壳

但如果目标是“顶级大项目”的细节体验与 UI 完成度，当前还存在以下缺口：

- 页面骨架已经统一，但细节交互还不够“专业工作台化”
- 结果阅读仍偏平铺，缺少“摘要层 -> 依据层 -> 原始层”的渐进披露
- 公网远勘与深度远勘的结果块已经可用，但还不够像“专业研判中枢”
- 内网扫描的结构化输出可读性还可以继续提升
- 视觉系统还停留在统一风格阶段，尚未进入“高级、克制、可信、带一点未来感”的成品状态

本轮仅继续修改远勘智能体，不影响其他模块。

## 设计目标

- 仅修改 `PentestPanel` 及其远勘相关渲染逻辑
- 将远勘工作台提升为“专业研判中枢 + 适量未来感氛围”
- 让功能体验和视觉体验同时拉满，而不是只美化外观
- 强化“结论优先、依据可追、原始输出下沉”的信息结构
- 保持低负担、少元素、可快速阅读的整体气质

## 外部资料提炼

参考资料方向：

- Flydash 提出 `Progressive Information Disclosure`
- Cabin 强调 `Map Decisions Before You Map Data`、`Lead with the answer, not the data`
- AufaitUX 强调 AI 仪表盘的 `Explainability / Adaptability / Reliability`

针对本项目，提炼为三条硬规则：

1. 第一屏只展示高价值决策元素
2. 所有复杂结果必须支持渐进披露
3. AI 结论必须附带“依据”和“下一步动作”语义

## 整体视觉方向

### 气质定位

不是炫技型指挥舱，也不是纯极简 BI 板，而是：

`专业研判工作台 + 克制未来感`

具体表现为：

- 更稳定的层级和留白
- 更少但更强的状态光效
- 更可信的结果卡和报告块
- 更有“中枢系统”感的操作与反馈

### 视觉语言

- 继续保留现有 Rich Glassmorphism 基底
- 降低大面积飘浮感，增加“板块锚定感”
- 强化边框、选中态、运行态、聚焦态的微差异
- 使用少量发光、渐变、状态脉冲来表达系统活性，而不是装饰

## 信息架构升级

### 总体阅读层级

所有模式统一为三层：

1. `摘要层`
2. `依据层`
3. `原始层`

其中：

- 摘要层：用于 3 秒内得到判断
- 依据层：用于说明为什么这么判断
- 原始层：用于深挖、核验、导出

### 公网远勘

公网远勘的升级重点不是加更多字段，而是把已有结果变成更像研判流：

- 第一屏：
  - 输入区
  - 执行控制
  - 状态与进度摘要
- 第二层：
  - 批量摘要条
  - 逐目标结论卡
  - 重点目标的结论块
- 第三层：
  - 证据与技术依据
  - 报告草稿

逐目标卡片升级为：

- 顶部：目标、风险、认证态
- 中部：用途/架构/风险/线索 4 宫格
- 底部：操作条
  - 查看重点目标
  - 带入深度远勘
  - 展开证据

### 深度远勘

深度远勘从“AI 对话页”升级成“专业分析工作台”：

- 左侧：侦查输入与控制
- 右侧：
  - 结构化结论
  - 依据与发现
  - 对话过程
  - 报告草稿

其中结构化结论要拆为固定模块：

- 重点结论
- 核心依据
- 可调证信息
- 后续侦查建议

这样用户不用先读长文，先看结论，再决定是否追依据。

### 内网扫描

内网扫描继续保持底层逻辑不变，但体验升级为：

- 第一层只看连接、上传、执行状态
- 第二层看结构化摘要
- 第三层才看详细表格和原始输出

结构化摘要区更像一个“扫描总览面板”，而不是直接堆表格。

## 关键交互升级

### 1. 渐进披露

通过折叠区、展开卡、摘要条实现：

- 默认只给结论
- 点击后查看依据
- 最后再看原始结果

### 2. 统一状态反馈

远勘工作台统一五种状态：

- idle
- loading
- success
- warning
- error

并统一显示在：

- Hero 状态摘要
- 局部状态条
- 全局通知条

### 3. 微交互

加入小而克制的交互反馈：

- 选中卡片的边框高亮与轻微抬升
- 运行中的状态脉冲
- 报告区与摘要区的平滑切换
- 关键按钮更明确的 hover/focus/disabled 层级

### 4. 可信 AI 体验

深度远勘强化“可信感”：

- 结论与依据分区
- 状态文案稳定
- 空态明确提示下一步
- 报告导出前就能预览最终成品结构

## 具体实现策略

仍以 `src/components/PentestPanel.tsx` 为主战场，不改其他模块。

本轮优先提炼以下可复用渲染块：

- 状态摘要条
- 折叠/展开卡
- 结果分层卡
- 结构化报告块
- 更高级的模式导航卡

如本轮结构明显变复杂，再考虑后续拆子组件，但本轮先不拆文件，避免范围外扩。

## 验证标准

- 仅 `PentestPanel` 相关逻辑变化
- 公网远勘的逐目标分析更像“研判卡”而不是普通结果卡
- 深度远勘的结论与依据分层清晰
- 内网扫描首页更简洁，原始输出下沉
- 全局 UI 更统一、更高级、更可信
- `GetDiagnostics` 无错误
- `npm run build` 通过
