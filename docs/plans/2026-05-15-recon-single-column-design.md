# 远勘结果区单列收敛设计

## 背景

当前远勘智能体已经具备较完整的结果层和报告层，但结果区存在明显的空间压力：

- 公网远勘同时并排展示“逐目标结论”和“重点目标/技术细节”
- 深度远勘同时并排展示“输入区”和“结构化结论/对话/报告”
- 当结果内容、对话内容、报告内容都比较长时，横向和纵向都会变得拥挤

用户明确反馈：`最后出来的结果界面都放不下`

因此本轮核心不是继续加模块，而是把结果阅读路径收成更稳的单列工作流。

## 设计目标

- 只修改远勘智能体
- 解决结果区“放不下”的问题
- 将主要阅读路径收敛为主内容单列
- 保持当前已经建立的层级、动画和成品感

## 总体策略

将结果区从“多块并排争抢空间”改为“单列主阅读流”。

单列不意味着信息减少，而是：

- 核心结果先出现
- 次级结果顺序下沉
- 技术细节默认折叠
- 长报告继续放在下方

## 公网远勘

### 当前问题

- 左右双栏导致注意力切碎
- 逐目标列表和重点目标详情互相抢空间
- 技术细节块会进一步挤压报告区

### 新布局

公网远勘改为以下顺序：

1. 批量摘要
2. 概览指标
3. 研判概览
4. 逐目标结论列表
5. 当前选中目标详情
6. 技术细节
7. 报告草稿

### 关键变化

- 逐目标列表仍保留，但不再和详情并排
- 点击目标后，详情区在列表下方显示
- 技术细节进一步折叠，减少默认占用
- 报告区保持在页面底部，阅读更连贯

## 深度远勘

### 当前问题

- 左右双栏在长对话、长报告场景下明显拥挤
- 输入区和结果区并排后，结果阅读宽度不足

### 新布局

深度远勘改为以下顺序：

1. 侦查输入
2. 工作重点
3. 结构化结论
4. 对话记录
5. 报告草稿

### 关键变化

- 输入区上移为流程起点
- 结构化结论优先于对话
- 对话和报告采用更强的高度约束与滚动
- 页面整体更像一条专业工作流

## 内网扫描

内网扫描已经相对接近单列结构，本轮只做收口：

- 步骤状态
- 操作区
- 扫描摘要
- 明细
- 原始输出

保持不做大改，只继续减少并排压力。

## 技术策略

- 继续以 `src/components/PentestPanel.tsx` 为主
- 如有必要，仅增加少量远勘布局 helper
- 报告预览区优先改成更保守的单列显示
- 保留此前的克制专业动画，但不为布局增加复杂度

## 验证标准

- 结果区明显更容易放下
- 公网远勘和深度远勘不再出现“主要内容互相挤压”
- 重点结果更容易连续阅读
- 报告区更容易完整浏览
- `GetDiagnostics` 无报错
- `npm run build` 通过
