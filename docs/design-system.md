# FUXI Server Forensics 设计系统

## 1. 设计理念

### 1.1 设计风格
- **参考**：Trae Solo 3 全新UI设计
- **风格**：现代化、专业、简洁、高效
- **目标**：打造一个既美观又实用的服务器分析与取证工具界面

### 1.2 设计原则
- **一致性**：统一的设计语言和视觉元素
- **清晰度**：清晰的信息层次和导航结构
- **响应式**：适配不同屏幕尺寸
- **可访问性**：确保界面易于理解和操作
- **性能**：优化视觉效果，确保流畅体验

## 4. 组件系统

### 4.1 按钮
- **主要按钮**：蓝色背景，白色文字，圆角4px

## 7. 代码规范

### 7.1 命名规范
- **组件**：PascalCase
- **函数**：camelCase
- **变量**：camelCase
- **常量**：UPPER_CASE
- **样式类**：kebab-case

### 7.2 文件结构
- **组件**：src/components/
- **样式**：src/styles/
- **工具**：src/lib/
- **配置**：src/config/

### 7.3 样式规范
- **使用 Tailwind CSS**：优先使用 Tailwind 工具类
- **自定义样式**：在 index.css 中定义
- **响应式设计**：使用 Tailwind 响应式类

## 8. 实现指南

### 8.1 组件实现
- **拆分组件**：将大型组件拆分为更小、更可维护的组件
- **状态管理**：使用 Zustand 管理状态
- **动画**：使用 Framer Motion 实现动画效果

### 8.2 性能优化
- **组件懒加载**：使用 React.lazy 和 Suspense
- **代码分割**：使用 Vite 的代码分割功能
- **虚拟滚动**：长列表使用虚拟滚动

### 8.3 可访问性
- **语义化 HTML**：使用正确的 HTML 标签
- **键盘导航**：支持键盘导航
- **屏幕阅读器**：添加适当的 ARIA 标签
